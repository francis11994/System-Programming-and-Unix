Extra Credit

You can get up to 10% extra credit on each of the spellcheck and myls problems by freeing all of the memory that was allocated dynamically (i.e., via malloc) during execution.

You can use Valgrind to determine whether you have freed all dynamically allocated memory. If you have, Valgrind will print

HEAP SUMMARY:
in use at exit: 0 bytes in 0 blocks
...
All heap blocks were freed -- no leaks are possible

In order to get the extra credit, your program must satisfy the following two conditions for all test inputs:

  1. all dynamically allocated memory is freed; and
  2. there are no memory errors.

Both these properties will be checked using Valgrind.

You should submit your programs separately for extra credit, using a different turnin assignment name, as indicated above. For each of these problems, your score will be the bigger of the score on your regular submission and (if submitted) your extra-credit submission.

The reason for setting up the extra credit separately is to protect you if, for any reason, there are any bugs in your memory-freeing code. This way, you can submit one set of files for "regular" credit and (possibly) a different set of files for extra credit; and you will still get credit for the "regular" submission even if there are problems in the extra-credit submission. Of course, you are free to submit the same set of files for both the regular and the extra credit submission.

NOTE: If you want your code to be submitted for extra credit, you must submit it as an extra-credit turnin.

For each of the two problems mentioned, your extra credit will be 10% of your score for the problem. Thus, if you get 82 out of 100 for a problem and your extra credit submission meets the two requirements mentioned above, you will get an additional 8.2 points (10% of 82), for a total score of 90.2.
