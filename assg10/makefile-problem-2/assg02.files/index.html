The purpose of this assignment is to get you started with writing some simple C programs, and in the process getting some experience with the preprocessor directive #include, the I/O routines printf and scanf, and some other library routines.

For some of the problems below, you need to read in integer values from stdin. You’ve seen in class how you can use the library function scanf () to do this. Remember %d is the code to use in the format string to read a decimal (integer) number. (For example, scanf (“%d”,&num);)

General Requirements

  1. Your C code should adhere to the coding standards for this class. Points will be subtracted from your score if your code does not adhere to these standards.
  2. Your programs should indicate whether or not they executed without any problems via their exit status, i.e., the value returned by the program when it terminates:

  1. Under bash you can check the exit status of a command or program cmd by typing the command "echo $?" immediately after the execution of cmd. A program can exit with status n by executing "exit( n)" anywhere in the program, or by having main() execute the statement " return(n) ".

Important Grading Notes

Be sure to name your files EXACTLY as specified on the page for each problem. (In this case isFib.c, sumSquares.c, and sumDigits.c) If you don't name the files correctly they won't get graded. Also, remember that Unix is case sensitive. "isfib.c" is NOT the same as "isFib.c"! Remember that your programs are required to compile without any warnings when you use the -Wall option in gcc. If you get warnings, fix them before you turn in your program. Lastly, take advantage of the given executable solutions. Your output to stdout should match them exactly. DON'T add prompts like "Please enter a digit". This will cause your program to fail the test cases. If you have any questions what the output should be, run the given executables.
